# FocusLock —— 硬件级番茄时钟 🍅

> 用硬件约束代替意志力，通过物理设备帮助用户建立健康的工作-休息节奏。

FocusLock 是一款基于 **ESP32-S3** 开发的专业级桌面生产力硬件。它不仅仅是一个计时器，而是能够在你的专注时间结束时，**自动向电脑发送组合键（如 Win+L 或 Cmd+L）强制锁屏**，强制你离开屏幕休息，从而保护视力和颈椎。

## ✨ 核心特性

- 🔒 **物理级防沉迷**：工作结束时通过 USB HID 模拟键盘强制电脑锁屏。
- 🕒 **硬件 RTC 精准计时**：内置 **DS3231** 硬件时钟，断电也能保存时间，OLED 实时显示当前时刻。
- 🔊 **声光双重提醒**：通过 WS2812 RGB 灯和有源蜂鸣器提供直观的状态反馈。
- 🥧 **可视化 UI**：双色 OLED 屏幕显示，具备实心饼图进度条和 24 小时制时钟。
- 📱 **Web 自动同步**：打开配置网页即可自动将电脑时间同步至硬件 RTC。
- ⚙️ **高度可配置**：支持自定义工作/休息时长、任意多键锁屏组合、蜂鸣器开关等。

## 🛠 硬件需求与连接 (针对 ESP32-S3)

**核心组件**：
- **主控**：ESP32-S3 (推荐 YD-ESP32-S3)
- **显示**：0.96寸 I2C OLED (SSD1306)
- **时钟**：DS3231 RTC 模块
- **声音**：5V 有源蜂鸣器 (推荐配合 S8550 PNP 驱动模块)
- **灯光**：WS2812B RGB LED (GPIO 8)
- **交互**：微动开关 (GPIO 0)

### 引脚分配表 (Pinout)

| 外设 / 接口 | ESP32 引脚 | 说明 |
| :--- | :--- | :--- |
| **I2C SDA** | **GPIO 41** | 连接 OLED 和 DS3231 的 SDA (并联) |
| **I2C SCL** | **GPIO 42** | 连接 OLED 和 DS3231 的 SCL (并联) |
| **Buzzer** | **GPIO 10** | 蜂鸣器信号线 (低电平触发) |
| **Button** | **GPIO 0** | 信号线，按下时接地 |
| **RGB LED** | **GPIO 8** | 信号线 |

> **⚠️ 硬件连接建议**
> 1. **电源**：OLED 和 RTC 建议接 **3.3V**；WS2812 和蜂鸣器建议接 **5V** (保证音量和亮度)。
> 2. **蜂鸣器防长鸣**：对于 5V PNP 驱动模块，请在蜂鸣器 **Signal (S)** 针脚与 **VCC (5V)** 之间跨接一个 **1kΩ 电阻**。
> 3. **共地**：所有模块的 GND 必须全部连接到开发板的 GND。

## 🚀 编译与烧录指南

本项目基于最新的 **ESP-IDF v6.0.1** 开发。

### 1. 准备环境
确保你已经安装了 ESP-IDF，并在终端中激活它：
```bash
# 路径请替换为你本机的实际安装路径
source ~/esp/esp-idf/export.sh
```

### 2. 编译与烧录
将开发板的 `USB` (Native) 接口连接到电脑，然后执行：
```bash
idf.py build flash monitor
```

## 🎮 使用说明

### 状态指示
- **工作 (WORK)**：OLED 显示倒计时，LED **常亮绿色**。
- **提醒 (WARNING)**：倒计时最后 30 秒，LED **黄色呼吸**，蜂鸣器慢鸣 3 声。
- **休息 (REST)**：发送锁屏指令，LED **常亮红色**，蜂鸣器急促鸣叫 5 声。
- **暂停 (PAUSE)**：LED **常亮蓝色**。

### 时间同步
连接至名为 **`FocusLock_Config`** 的 Wi-Fi 后，打开浏览器访问 `192.168.4.1`。网页会自动获取您的电脑时间并下发给 DS3231 模块，实现自动对时。

## 🏗 系统架构

- `i2c_manager`：集中化 I2C 总线管理，具备总线复位 (Bus Reset) 逻辑。
- `rtc_service`：管理 DS3231 通信及 BCD 码转换。
- `buzzer_service`：提供非阻塞的异步鸣叫序列。
- `oled_service`：基于 `u8g2` 的 UI 渲染，适配双色屏布局。
- `pomodoro_engine`：核心状态机，通过 FreeRTOS 队列驱动各服务协同。
