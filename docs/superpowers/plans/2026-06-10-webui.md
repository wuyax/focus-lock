# Web Management Interface Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement an on-demand Wi-Fi AP, DNS Captive Portal, and HTTP Web Server to allow users to configure FocusLock settings via a browser when in ADMIN mode.

**Architecture:** A `network_service` task monitors the `status_queue`. When `STATE_ADMIN` is active, it starts the Wi-Fi AP, DNS server, and HTTP server. When exiting `STATE_ADMIN`, it shuts them down. The HTTP server provides REST APIs (`/api/config`, `/api/stats`) and serves an embedded static Single Page Application (SPA).

**Tech Stack:** C, ESP-IDF (esp_wifi, esp_http_server, lwip, cJSON), HTML/JS.
